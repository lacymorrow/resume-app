### Day 16 (Tuesday) — Dev Tip: Database Timestamps
**@lacybuilds:**
> Dev tip: Use timestamps instead of booleans in your database.
>
> `activeAt: timestamp` instead of `isActive: boolean`
>
> You get the boolean check (is it null?) PLUS when it happened. Free audit trail.
>
> This is how every table in Shipkit works.

### Day 24 (Wednesday) — Drizzle ORM
**@lacybuilds:**
> Why I picked Drizzle ORM over Prisma for Shipkit:
>
> - Type-safe SQL (not an abstraction over SQL)
> - Migrations that make sense
> - Drizzle Studio for visual DB management
> - Faster cold starts (no binary engine)
>
> `bun run db:generate && bun run db:migrate` — done.

**@shipkit-io:**
> Shipkit uses Drizzle ORM for type-safe database operations.
>
> Schema changes: edit -> generate -> migrate. That's it.
>
> No binary engine. No cold start penalty. Just SQL that TypeScript understands.
